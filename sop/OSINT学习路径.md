## 三阶段规划

### Phase 1 · 被动侦察（零API Key，纯公开信息）
**目标：** 不发送任何探测包，仅利用公开数据库和信息源

| 模块 | 技术 | 落地到ToolPilot |
|:-----|:------|:---------------|
| 1.1 | DNS记录分析（A/AAAA/MX/NS/TXT） | 发现工具域名背后的基础设施 |
| 1.2 | Certificate Transparency（crt.sh） | 发现工具的所有子域名和部署 |
| 1.3 | Web技术栈识别（响应头/HTML分析） | 自动标记工具用了什么技术 |
| 1.4 | WHOIS/RDAP查询 | 了解工具的所有者和注册信息 |
| 1.5 | 搜索引擎高级语法（site:/inurl:） | 发现隐藏的Demo/测试站点 |

**预计耗时：** 3-5天，从零开始

### Phase 2 · 主动测绘（注册免费API）
**目标：** 利用Shodan/FOFA/Censys等平台的免费额度做资产发现

| 模块 | 平台 | 免费额度 | 用途 |
|:-----|:-----|:--------:|:-----|
| 2.1 | FOFA (fofa.info) | 注册送积分 | 国内AI服务资产测绘 |
| 2.2 | Shodan | 免费账户每月配额 | 全球暴露服务扫描 |
| 2.3 | Censys | 免费API 250查询/月 | 备选数据源 |
| 2.4 | SecurityTrails | 免费API 50查询/月 | 历史DNS数据 |

### Phase 3 · 自动化狩猎融合
**目标：** 将OSINT能力集成到现有狩猎管道中，自动补充工具信息

| 模块 | 功能 | 优先级 |
|:-----|:------|:------|
| 3.1 | 狩猎→自动域名发现→crt.sh查询→子域名列表 | P0 |
| 3.2 | 子域名→存活检测→截屏→技术识别 | P0 |
| 3.3 | FOFA查询→暴露服务列表→版本识别 | P1 |
| 3.4 | 狩猎数据→Shodan查询→全球部署热度分析 | P1 |
| 3.5 | 自动生成"全球部署报告"卡片→内容素材 | P2 |
